# 仓库历史已重写 — 同步说明

我已对仓库历史执行了不可逆的重写（移除了调试/临时文件），并已强制推送到 `origin/main`。所有协作者必须重置本地克隆以避免冲突：

推荐步骤（覆盖本地分支，**会丢失未提交/未保存的本地更改**）：

```bash
# 1. 保存或暂存本地未提交更改（如果需要）
git stash push -m "保存本地更改（在重写历史前）"

# 2. 获取远端最新引用
git fetch origin

# 3. 重置本地分支到远端主分支
git checkout main
git reset --hard origin/main

# 4. 清理未跟踪文件（可选）
git clean -fdx
```

说明：如果你有基于原历史的本地分支需要保留，请先备份分支或将变更导出为 patch。
